## Changes Made

### 1. Updated `BreakPlan` Interface

**File**: `packages/types/src/break.ts`

**Added Fields**:

**File**: `packages/types/src/break.ts`

### 3. Updated OpenAPI Specification

**File**: `docs/api/openapi.yaml`

#### 3.1 Updated `BreakPlan` Schema

Added the same fields to the OpenAPI schema:
- `feeling` (string)
- `chosen` (boolean)
- `order` (number)
- `updated_at` (date-time)

All fields are marked as required.

#### 3.2 Updated POST `/break/plans` Response

**Before**: Returned a single `BreakPlan`

**After**: Returns `BreakPlanListResponse` (array of BreakPlan)

**Changes**:
- Updated description to mention multiple plans are generated
- Response now returns array of plans
- Added `429` (Rate limit exceeded) response

#### 3.3 Added PATCH `/break/plans/{id}/chosen` Endpoint

**New Endpoint**: `PATCH /break/plans/{id}/chosen`

**Purpose**: Mark a break plan as chosen

**Parameters**:
- `id` (path parameter) - Break plan ID (UUID)

**Responses**:
- `200` - Success (returns updated BreakPlan)
- `400` - Invalid request
- `401` - Unauthorized
- `403` - Forbidden (plan doesn't belong to user)
- `404` - Break plan not found
- `500` - Internal server error

## API Endpoints Summary

### Break Plans

1. **GET `/break/plans`**
   - List all break plans for authenticated user
   - Returns: `BreakPlan[]`

2. **POST `/break/plans`**
   - Create multiple break plans (generated by LLM)
   - Request: `CreateBreakPlanRequest`
   - Returns: `BreakPlan[]` (array of created plans)
   - Rate limited: 10 requests/hour per user

3. **PATCH `/break/plans/{id}/chosen`** ⭐ NEW
   - Mark a break plan as chosen
   - Returns: `BreakPlan` (updated plan)
   - Authorization: Only plan owner can update

### History

1. **GET `/history`**
   - Get paginated history (filtered by `chosen: true`)
   - Query params: `cursor`, `limit` (default: 7)
   - Returns: `HistoryResponse`
